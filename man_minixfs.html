minixfs - Minix-Filesytem image manipulation tool. minixfs [global options] subcmd {img-file} [sub options]

minixfs is a tool to manipulate Minix filesystem images. Allows you to create arbitrary filesystem images from user space.

Normally it would be used for generating RAMDISK images for initrd. {img-file} Name of the Minix file system image

-f,--check (global option) Checks filesystem state and fails if a fsck is needed.

-q,--squash-ids (global option) Squash ids so all files are owned by root.

--help,--usage (global option) Prints the command usage

--version (global option) Print version information

--copyright (global option) Print copyright information

mkfs (sub command) Create an empty filesystem

-n namelen (mkfs, genfs option) Specify the maximum length of filenames. Currently, only allowable values are 14 and 30.

-i inodecount (mkfs, genfs option) Specify the number of inodes for the filesystem.

-1 (mkfs, genfs option) Make a Minix version 1 filesystem.

-2|-v (mkfs, genfs option) Make a Minix version 2 filesystem. This is the default.

-k (genfs option) Keep original raw data on disk (eg.: bootloader)

-s size (mkfs, genfs option) Specify the size of filesystem (kilobytes).

-B bootloader (mkfs, genfs option) Install bootloader file

dir (sub command) Display the contents of a directory path

stat (sub command) Display the inode details for a file

mkdir (sub command) Create a sub-directory

rmdir (sub command) Remove directories

unlink (sub command) Remove files

cat (sub command) Equivalent of the UNIX cat command.

extract (sub command) Extracts a file to the specified file name

readlink (sub command) Display a symbolic link

symlink (sub command) Create a symbolic link

hardlink (sub command) Create a hard link

file|dir|socket|pipe|char|block (sub command) Create a node or set file attributes

add (sub command) Copy a file from file-system to image.
